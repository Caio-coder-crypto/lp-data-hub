# Notas de design — LP DataHub (rebuild jun/2026)

## Brief do cliente (Caio)
- Tirar o lado PESSOAL ("Quem faz / Sou o Caio") — parece amador, "só um cara". Reposicionar pra MISSÃO/objetivo da empresa ("a gente vê muitas operações e descobriu...").
- Deixar ENXUTA.
- Frontend premium nível Nike/Adidas/DataCrazy.
- Objeto 3D que TRANSFORMA conforme rola a página.
- Mobile à altura (e fallback caso 3D não carregue).

## Anti-default (skill)
O default a evitar é exatamente "near-black + single acid-green" — que é a LP ATUAL. Diferenciação:
1. Assinatura = campo de partículas 3D scroll-morph (ruído→estrutura→crescimento).
2. Paleta DUAL verde(receita)/coral(vazamento) — literaliza o headline "de onde vem / por onde vaza". É o risco justificado.
3. Display = Clash Display (Fontshare), não o Sora SaaS-default.
4. Hero imersivo full-bleed, não headline+grid de screenshot.

## Tokens
- Base: #05080D ink / petróleo. Verde marca: #00C16A / #43E59A. Coral: #FF7E54.
- Display: Clash Display. Body: Inter. Utilitário/mono: Space Grotesk.

## 3D (Three.js CDN, ESM importmap)
- 1 BufferGeometry de N pontos (desktop ~5200, mobile ~2200), 5 shapes-alvo:
  esfera(hero) → nuvem(problema) → malha(método) → funil(solução) → hélice(crescimento).
- Morph por lerp CPU dirigido pelo scrollProgress; cor verde↔coral por uniform (coral perto do shape "nuvem/problema").
- ShaderMaterial: pontos redondos, additive glow, twinkle.
- Mobile: N menor, DPR<=2, sem parallax. reduced-motion: render estático (esfera), sem loop. sem WebGL: esconde canvas (gradiente do body fica).
